TIER 1 — THE BRAIN
🧠
OpenClaw Gateway
Agent Framework · Neb VPS
The orchestration layer. OpenClaw is an open-source framework that runs on the VPS as a systemd service. It manages sessions, routes messages between channels (Telegram, webchat), handles tool calls, memory, compaction, heartbeats, and cron jobs. Think of it as the nervous system — everything flows through here.
→ Claude API → OpenRouter ← Telegram ← Webchat Hosts: Neb VPS
  • Runs as openclaw.service (systemd, auto-restart)
  • Manages session files, JSONL logs, memory search (embeddings)
  • Dispatches tool calls: exec, browser, web_search, nodes, message
  • Heartbeat every 60min — checks SMS/email alerts, runs diagnostics
  • Cron engine: SMS scan, email scan, Telegram channel posts (4x/day)
  • Compaction: auto-summarizes when context exceeds 80K tokens
⚡
Anthropic Claude API
Primary LLM · Claude Sonnet 4.5
The "thinking" model. All M0RPH3US reasoning, code generation, and conversation goes through Claude. OpenClaw sends prompts, Claude returns responses + tool calls.
API Key: sk-ant-* ← OpenClaw prompts → Responses + tool calls
  • Default model: claude-sonnet-4-5
  • Opus available for complex tasks
  • Token managed by OpenClaw (session tokens)
  • Supports vision, code, extended thinking

SYSTEM SUMMARY

The Big Picture

The Construct is a 24/7 autonomous AI system built on open-source infrastructure. At its core:

1. Brain: Claude (Anthropic) does all the reasoning. It's the LLM that "thinks."
2. Nervous System: OpenClaw is the agent framework — it manages conversations, tools, memory, and routing. Runs on the VPS.
3. Body: The Nebuchadnezzar VPS (DigitalOcean) is where OpenClaw lives. 24/7 uptime, Linux, cron jobs, git operations.
4. Arms: APIs and tools — GitHub for code, Maton for Google services, Termux for phone control, Vercel for deployment.
5. Muscles: The Desktop node provides GPU power (RTX 5090) for ML training and local LLM inference when needed.
6. Eyes: SMS and email scanners watch for critical messages. The heartbeat checks system health every hour.
7. Memory: Markdown files (MEMORY.md, daily logs, WORKING_BUFFER.md) provide continuity across sessions. Semantic search via embeddings.

Everything connects through Tailscale's encrypted mesh network. No public ports are exposed. The AI can read emails, send texts, deploy websites, run ML experiments, manage finances, and monitor the business — all autonomously, 24/7.
